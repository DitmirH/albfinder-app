# Supabase setup for AlbFinder

## 1. Create a Supabase project

1. Go to [supabase.com](https://supabase.com) and create a project.
2. In **Project Settings → API**: copy the **Project URL** and **anon public** key (or **service_role** for the import script).

## 2. Run the migration

1. In Supabase: **SQL Editor → New query**.
2. Paste the contents of `supabase/migrations/001_records_table.sql`.
3. Click **Run**. This creates the `records` table, RLS policy, and RPCs.

## 3. Environment variables

Add to your `.env` (and to Netlify env vars for production):

```env
VITE_SUPABASE_URL=https://YOUR_PROJECT_REF.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_key_here
```

For the seed script (local only), you can use the same anon key or the **service_role** key:

```env
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key   # optional, for seed script
```

## 4. Seed the database from data.json

With `public/data.json` present (e.g. generated by `python convert.py` from your CSV):

```bash
node scripts/seed-supabase.js
```

Or with explicit env:

```bash
VITE_SUPABASE_URL=https://xxx.supabase.co SUPABASE_SERVICE_ROLE_KEY=xxx node scripts/seed-supabase.js
```

## 5. Run the app

With `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` set, the app will use Supabase for data (paginated list and detail). Without these env vars, the app falls back to loading `/data.json` (static file).
